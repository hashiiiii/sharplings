This confirms with concrete diagnostics, rather than expectation, that these three are `Not supported` on Mono. Their behavior on the CoreCLR Desktop Player — a real .NET runtime that supports all three — is exactly the open question the deferred CoreCLR player experiment (Track A, Tasks 3-6) would answer; the probes are already written and guarded for that run. Also hardened `ProbeRuntimeRunner`: the whole reflection scan now sits inside a `try`/`finally` with `Application.Quit(0)` in the `finally`, so a scan-time exception (e.g. `ReflectionTypeLoadException` under managed stripping) can no longer hang a headless player.
